CSS-布局经验

#参考资料：learnlayout网站

#一、响应式布局：

###1.特点：
（元素）位置变化、（元素）尺寸变化、字体（大小）变化

###2.方法：
诸多技术的综合：自适应+弹性布局

###3.优点：
一套代码通用所用设备，所有设备上完美显示！

###4.缺点：
代码量大，考虑的问题多

#二、自适应布局

###1.特点：
（元素）位置变化、（元素）尺寸固定、字体（大小）固定

###2.方法：
@media查询，每个尺寸范围内，对应一套css样式

###3.优点：
屏幕改变到临界点时，元素位置改变，做出最佳显示效果

###4.缺点：

  • 元素和屏幕之间不能等比例缩放；
  • 元素本身宽高之间不能等比例缩放；
  • 为适应多种规格屏幕，需要做多套css样式；

#三、弹性布局：（动态rem布局、vw布局）

###1.特点：
（元素）位置固定，（元素）尺寸变化，字体（大小）不变/变化

###2.方法：

  1. 思路：
  • 手机尺寸不同，说得是宽度不同，窄手机元素小，宽手机元素大，即让元素尺寸跟随屏幕宽度变化，
  • px、em和rem等固定单位都是不行的，vw单位可以，因为vw和手机尺寸直接关联！
  • rem和html的font-size有关系，如果让html的font-size和屏幕宽度建立联系，那么rem和屏幕宽度也就关联上了！
  1. rem布局：

  • 核心：屏幕宽度 = 10rem

  • 实战：屏幕宽度为500px，元素1宽度为100px，元素2宽度为300px，如何用rem表示？

    • 元素1：
      px单位：100/500 500px
      rem单位：100/500       10rem
    • 元素2：
      px单位：300/500 500px
      rem单位：300/500       10rem
  • 注意：太小的地方就不要用rem了，比如border的宽度，还有，body的font-size设置成16px就好了。
  1. vw布局：
  • 核心：屏幕宽度 = 100vw
  • 实战：屏幕宽度为500px，元素1宽度为100px，元素2宽度为300px，如何用rem表示？
    • 元素1：
      px单位：100/500 500px
      rem单位：100/500       100vw
    • 元素2：
      px单位：300/500 500px
      rem单位：300/500       100vw

###3.优点：
元素和屏幕之间等比例缩放，元素本身宽高之间等比例缩放

###4.缺点：

  • 动态rem布局每次加载需要刷新才起作用，电脑不行，手机可以，手机是一次性加载！
  • rem在电脑端的兼容性还不是太好，不过现在已经得到了很大的改善，查看兼容情况：
    兼容性查询

#四、流式布局：（百分比布局）

###1.特点：
（元素）位置固定、（元素）尺寸变化、字体（大小）固定

###2.方法：

  1. 使用百分比宽度（%）、绝对高度（px）
  2. 设置max-width、min-width
    （设置网页主体宽度为80%，min-width为960px）

###3.优点：
元素和屏幕之间等比例缩放

###4.缺点：

  • 元素本身宽高不能等比例缩放，因为宽度无法确定，高度没法跟着宽度走，
  • 当元素宽度被过度拉长或挤压时高度固定，会设计失真

#五、固定布局：

###1.特点：
（元素）位置固定、（元素）尺寸固定、字体（大小）固定

###2.方法：

  1. 使用绝对宽度（px）、绝对高度（px）
  2. 设置max-width、min-width

###3.优点：
简单、没兼容性问题

###4.缺点：
屏幕尺寸改变，网页不能自动缩放

#六、手机端布局：

  • mobile-first（先做手机版：推荐） && desktop-first（先做电脑版）
  • 某个元素在同一屏幕显示不同状态，由JavaScript去做，即JS控制元素的类/样式；
  • 某个元素在不同屏幕显示不同状态，由媒体查询去做，即符合条件时样式起作用，否则不起作用；

###方案1：响应式：一套样式表，样式表下载，符合查询结果的样式生效！

###方案2：响应式：多套样式表，样式表全部下载，符合查询结果的样式表生效！

###方案3：非响应式：后端判断显示设备是什么，userAgent？
是手机就下载手机页面，是电脑就下载电脑页面，这些资源在同一服务器上。

###方案4：非响应式：后端判断显示设备是什么，userAgent？
是手机就从xx服务器下载xx资源，是电脑就从yy服务器下载yy资源。
例子：（淘宝手机/电脑版，京东手机/电脑版）

CSS-布局之定位布局

#参考资料：《CSS权威指南》page：307页

#一、绝对定位：

  • 相对于包含块移动，偏移属性确定移动的距离；
  • 原位置删除，新位置不会嵌入其他元素的内容；（脱离文档流）
  • 块级元素生成块级框，行内元素也是生成块级框；（形成BFC）
  • 包含块：body元素或position不为static的祖先元素

属性：

  • top—-“下移”
  • bottom—-“上移”
  • left—-“右移”
  • right—-“左移”

#二、相对定位：

  • 相对于原位置移动，原位置保留，偏移属性确定移动的距离；
  • 块级元素生成块级框，行内元素生成行内框

属性：

  • top—-“下移”
  • bottom—-“上移”
  • left—-“右移”
  • right—-“左移”

#三、固定定位：

  • 相对于视窗移动，偏移属性确定移动的距离；（与absolute相似）
  • 原位置删除，新位置不会嵌入其他元素的内容；（脱离文档流）
  • 块级元素生成块级框，行内元素也是生成块级框（形成BFC）

属性：

  • top—-“下移”
  • bottom—-“上移”
  • left—-“右移”
  • right—-“左移”

#四、其他知识：

  • 根元素html的包含块是视窗大小的矩形
  • position样式值是relative或static时，包含块由最近的块级框、表单元格或行内块祖先框的内容边界构成
  • position样式值是absolute时，包含块由最近的position值不是static的祖先元素构成，
    当祖先元素是块级元素时，包含块是元素的内边距边界，
    当祖先元素是行内元素时，包含块是元素的内容边界。
  • 如果没有祖先元素，元素的包含块定义为初始包含块
  • 元素绝对定位后，会为其后代元素建立一个包含块
    （position不为static的都可做包含块）
  • 绝对定位元素相对于包含块移动，包含块是正常流的一部分，
    当文档流滚动时，包含块随之滚动，定位元素也随之滚动
  • 非替换元素的绝对定位时
    • left + margin-left + border-left + padding-left + width + padding-right + border-right + margin-right
      = 包含块的width
      • top + margin-top + border-top + padding-top +height + padding-bottom + border-bottom + margin-bottom
        = 包含块的height

CSS-布局之float布局

#参考资料：《CSS权威指南》page：288页

我的一点总结：

  • 浮动元素：使元素脱离文档流，元素自动生成一个块级框（BFC）。
  • 脱离文档流的意思就是，正常的文档流中丢失了它的位置。
  • 浮动的框可以向左或向右（left、right）移动，直到他的外边缘碰到包含框或另一个浮动框为止。
  • 三步走：浮动元素抬起来，下面元素顶上去，浮动元素落下来
  • 父元素在高度自适应的情况下，由于浮动丢失了子元素的高度，父元素就无法被撑开（塌陷），所以 导致父元素的背景、边框等属性无法生效，也会使后面的元素顶上来。
  • 我们使用浮动一般是为了让块级元素实现行级元素的特性（类似inline-block），但也不希望浮动元素影响后面的元素位置
  • 当浮动失去文档流，使后面的元素顶上来以后，一般有三种方法可以处理：
    • 1、为父元素增加height属性，这样父元素就会显示边框、背景等，但这种情况是确定浮动元素 的高度时才这么做，但是实际中，可能不清楚高度哦
    • 2、为浮动元素后面的元素增加clear:both（clearfix清浮动），清除浮动效果，如果没有后面 元素的话，增加一个元素。
    • 3、为父元素设置overflow:hidden属性（BFC—–块格式化上下文）

  • float属性：

    • left—-“左浮动”
    • right—-“右浮动”
    • none—-“不浮动”

CSS-布局之flex布局

#参考资料：MDN文档，阮一峰的文章
阮一峰：Flex 布局教程：语法篇

随心追梦：Flexbox完整指南- A Complete Guide to Flexbox

巩固flex知识的小游戏1

巩固flex知识的小游戏2

#1、预备知识：

  • flex容器中默认存在两条轴，主轴（main axis）和交叉轴（cross axis）；
  • flex容器是flex container，每个项目称为flex item；
  • 主轴是水平方向时，每个flex item占据的主轴空间为main size，占据的交叉空间为cross size；
  • 主轴起点是main start，终点是main end；
  • 交叉轴起点是cross start，终点cross end ；
  • 制作flex盒子：.container{display: flex/inline-flex}
    块状元素用flex，行内元素 用inline-flex；
  • 设成了flex盒子后，子元素的float、clear、vertical-align属性都会失效！

#2、flex容器（flex container）的属性

  1. flex-direction —-“主轴的方向”

    属性：

    • row—-“水平方向，起点在左” 默认值
    • row-reverse —-“水平方向，起点在右”
    • column—-“垂直方向，起点在上”
    • column-reverse —-“垂直方向，起点在下”

  2. flex-wrap —-“是否换行”

    属性：

    • nowrap—-“不换行” 默认值
    • wrap—-“换行”
    • wrap-reverse —-“换行，首行在下”

  3. justify-content —-“容器里项目 主轴对齐方式”

    属性：

    • flex-start —-“靠向起点” 默认值
    • flex-end —-“靠向终点”
    • center —-“居中”
    • space-between —-“两端对齐”
    • space-around —-“等间隔对齐”

  4. align-items —-“容器里项目 交叉轴对齐方式”

    属性：

    • strentch —-“延伸” 默认值
    • flex-start —-“靠向起点”
    • flex-end —-“靠向终点”
    • center —-“居中”
    • baseline —-“项目第一行文字的基线对齐” 不常用

  5. align-content —-“项目在多根轴线对齐方式”

    属性：

    • stretch—-“延伸” 默认值
    • flex-start —-“靠向交叉轴起点”
    • flex-end —-“靠向交叉轴终点”
    • center—-“居中”
    • space-between —-“交叉轴两端对齐”
    • space-around —-“交叉轴等间隔对齐”

  6. flex-flow —-“方向 换行” 简写

#3. flex项目（flex item）的属性：

  1. order —-“排列顺序，值小靠前”

  2. align-self —-“单个项目在交叉轴方向上的对齐方式”

  • 默认值auto，表示继承父元素的align-items属性
  1. flex-basis —-“main-size值”
  • flext-size未设定值：默认值为auto
  1. flex-grow —-“增长比例（空间过多时）”
  • 默认值为0，空间有多余时，项目也不放大；设置为1，项目可能放大
  1. flex-shrink —-“收缩比例（空间不够时）”
  • 默认值为1，空间不够时，项目自动缩小；设置为0，项目不会缩小
  1. flex —-“增长比例 收缩比例 默认值”简写

#参考资料：张鑫旭的文章

#一、自己写的小诗一首：

    《位置层叠，谁上谁下》
不同层叠上下文，寻根问祖祖为主；
同一层叠上下文，层叠水平是根由；
层叠水平不同时，七层定律要遵守；
层叠水平相同时，后来居上登高楼。

#二、判断方法：

  • 如何判断两个位置发生层叠的元素，谁在上面谁在下面呢？

  1. 如果两元素不在同一层叠上下文中，那么寻找各自的祖先元素，直到祖先元素同处一个层叠上下文中，然后比较祖先元素的层叠水平

    ChildA和ChildC，不在同一层叠上下文，但是ParentA和ParentB在同一层叠上下文Root中，比较ParentA和ParentB就好了

  2. 如果两个元素在同一个层叠上下文中，那么直接比较他们的层叠水平就好了

    ChildA和ChildB，在同一个层叠上下文ParentA，比较他们自身的层叠水平就好了；

  3. 当层叠水平不同时，按照七层定律做最终判断，判断谁上谁下

  4. 当层叠水平相同时，写在后面的元素覆盖写在前面的元素

CSS-BFC

#参考资料：方应杭的视频

#一、什么是BFC？
块格式化上下文（Block Formating Context, BFC）是web页面的可视化CSS渲染的一部分，是布局过程中生成块级盒子的区域，也是浮动元素与其他元素的交互限定区域。

#二、BFC有什么用？
实际开发中遇到的4个问题，都可以用BFC解决

1. 有两个div元素，是兄弟关系，  
    都设置了外边距，此时外边距是两者之和吗？当然不是，因为发生了外边距合并。  
2. 有两个div元素，是父子关系，  
    如果父元素没有设置border或者padding，子元素设置了margin-top，最终的显示效果是子元素margin超出父元素区域，使父元素移动。
3. ul中li元素全部浮动后，ul元素会出现塌陷，塌陷的意思是高度没有了。
4. 如果一个元素浮动脱离了文档流，下面的元素就会跑上去，但是会被浮动的元素遮盖住。

#三、怎么创建BFC？

1. 根元素或包含根元素的元素
2. 浮动元素  
    （元素的 float 不是 none）
3. 绝对定位元素  
    （元素的 position 为 absolute 或 fixed）
4. 行内块元素  
    （元素的 display 为 inline-block）
5. overflow 值不为 visible 的块元素
6. display 值为 flow-root 的元素  
    （无副作用，好用）
7. display为 flex 或 inline-flex元素的直接子元素  
    （flex-items拥有BFC，而不是flex-container）
8. display为 grid 或 inline-grid 元素的直接子元素  
    （grid-items拥有BFC，而不是grid-container）
9. 表格单元格（元素的 display为 table-cell，HTML表格单元格默认为该值）
10. 表格标题  
    （元素的 display 为 table-caption，HTML表格标题默认为该值）
11. 匿名表格单元格元素        
   （元素的 display为 table、table-row、 table-row-group、table-header-group、table-footer-group
（分别是HTML table、row、tbody、thead、tfoot的默认属性）或 inline-table）
12. contain 值为 layout、content或 strict 的元素
13. 多列容器  
    （元素的 column-count 或 column-width不为auto，包括column-count为1）
14. column-span 为 all 的元素始终会创建一个新的BFC，即使该元素没有包裹在一个多列容器中（标准变更，Chrome bug）。

#四、慎用BFC！

  • 创建BFC的方法大都有副作用，因为那些方法本身有其自身的设计目的，比如浮动、绝对定位还有超出隐藏等等，
  • 虽然 display: flow-root; 没有副作用，但是兼容性暂时不好，故而BFC要慎用！
  • 需要BFC解决的问题，都有其他更好的办法解决！比如浮动塌陷可以用clearfix清浮动解决。
