CSS-布局之grid布局

#参考资料：MDN文档，知乎-若愚-文章

若愚：CSS Grid 系列(上)-Grid布局完整指南

#1、预备知识：

  1. 网格线：分界线组成网格，可以是水平的也可以是垂直的
  2. 网格单元：网格单元是网格中最小的单位，类似表格中的单元格
  3. 网格轨道：两个相邻网格线之间的空间，可横可竖
  4. 网格区域：网格项目向行或列方向扩展一个或多个单元，并且会创建一个网格区域
  5. 网格嵌套：一个网格项目也可以成为一个网格容器，嵌套的网格和它的父级没有关系

#2、grid容器（grid container）的属性：

  1. grid-template-areas —-“布局可视化”

  2. grid—-“简写：先行后列”

    • grid-template-rows —-“行高”
    • grid-template-columns —-“列宽”

  3. grid-gap —-“简写：先行后列”

    • grid-column-gap —-“列间距”
    • grid-row-gap —-“行间距”

  4. justify-content —-“容器里网格 行轴对齐方式”
    align-content —-“容器里网格 竖轴对齐方式”

    属性值：

    • start —-“左对齐、上对齐”
    • end —-“右对齐、下对齐”
    • center —-“居中对齐”
    • space-around —-“项目的四周空间”
    • space-between —-“项目的相邻空间”
    • space-evenly —-“项目的均匀空间”

  5. justify-items —-“项目里内容 行轴对齐方式”
    align-items —-“项目里内容 竖轴对齐方式”

    属性值：

    • start —-“”
    • end —-“”
    • center —-“”
    • stretch —-“”

  6. grid-auto-columns —-“隐式网格列宽”
    grid-auto-rows —-“隐式网格行高”

  7. grid-auto-flow —-“项目自动放置”

  8. repeat—-“重复宽度”

  1. fr—-“自由空间比例单位”
  1. minmax(min, max) —-“参数：最小值，最大值”

#3. grid项目（grid item）的属性：

  1. grid-area —-“项目区域命令”

  2. grid-column —-“简写：start+end”
    grid-row —-“简写：start+end”

    • grid-column-start —-“项目竖轴上起始线”
    • grid-column-end —-“项目竖轴上结束线”
    • grid-row-start —-“项目行轴上起始线”
    • grid-row-end —-“项目行轴上结束线”
  1. justify-self —-“单个儿项目里内容行轴对齐方式”
    align-self —-“单个儿项目里内容竖轴对齐方式”

#4. 使用习惯：

  1. 搭配方案1：
    容器的grid-template
    项目的grid-area
  1. 搭配方案2：
    容器的grid-template-columns、grid-template-rows，简写属性grid
    项目的grid-row、grid-column

CSS-BFC

#参考资料：方应杭的视频

#一、什么是BFC？
块格式化上下文（Block Formating Context, BFC）是web页面的可视化CSS渲染的一部分，是布局过程中生成块级盒子的区域，也是浮动元素与其他元素的交互限定区域。

#二、BFC有什么用？
实际开发中遇到的4个问题，都可以用BFC解决

1. 有两个div元素，是兄弟关系，  
    都设置了外边距，此时外边距是两者之和吗？当然不是，因为发生了外边距合并。  
2. 有两个div元素，是父子关系，  
    如果父元素没有设置border或者padding，子元素设置了margin-top，最终的显示效果是子元素margin超出父元素区域，使父元素移动。
3. ul中li元素全部浮动后，ul元素会出现塌陷，塌陷的意思是高度没有了。
4. 如果一个元素浮动脱离了文档流，下面的元素就会跑上去，但是会被浮动的元素遮盖住。

#三、怎么创建BFC？

1. 根元素或包含根元素的元素
2. 浮动元素  
    （元素的 float 不是 none）
3. 绝对定位元素  
    （元素的 position 为 absolute 或 fixed）
4. 行内块元素  
    （元素的 display 为 inline-block）
5. overflow 值不为 visible 的块元素
6. display 值为 flow-root 的元素  
    （无副作用，好用）
7. display为 flex 或 inline-flex元素的直接子元素  
    （flex-items拥有BFC，而不是flex-container）
8. display为 grid 或 inline-grid 元素的直接子元素  
    （grid-items拥有BFC，而不是grid-container）
9. 表格单元格（元素的 display为 table-cell，HTML表格单元格默认为该值）
10. 表格标题  
    （元素的 display 为 table-caption，HTML表格标题默认为该值）
11. 匿名表格单元格元素        
   （元素的 display为 table、table-row、 table-row-group、table-header-group、table-footer-group
（分别是HTML table、row、tbody、thead、tfoot的默认属性）或 inline-table）
12. contain 值为 layout、content或 strict 的元素
13. 多列容器  
    （元素的 column-count 或 column-width不为auto，包括column-count为1）
14. column-span 为 all 的元素始终会创建一个新的BFC，即使该元素没有包裹在一个多列容器中（标准变更，Chrome bug）。

#四、慎用BFC！

  • 创建BFC的方法大都有副作用，因为那些方法本身有其自身的设计目的，比如浮动、绝对定位还有超出隐藏等等，
  • 虽然 display: flow-root; 没有副作用，但是兼容性暂时不好，故而BFC要慎用！
  • 需要BFC解决的问题，都有其他更好的办法解决！比如浮动塌陷可以用clearfix清浮动解决。

CSS-选择器优先级

#参考资料：《CSS权威指南》page：70页

#一、CSS选择器优先级如何确定？

  1. 考虑权重：有!import > 没有!import

  2. 考虑位置和引入方式：内联样式 > 内部样式块 > 外部样式表
    后面出现 > 前面出现

  3. 考虑特殊性：高特殊性 > 低特殊性 > 没有特殊性
