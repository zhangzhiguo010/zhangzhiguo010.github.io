##二、Babel

官网：https://github.com/babel/babel-loader
命令：npm install -D babel-loader @babel/core @babel/preset-env webpack
版本：webpack 4.x | babel-loader 8.x | babel 7.x
配置：module: {
        rules: [
            {
            test: /\.m?js$/,
            exclude: /(node_modules|bower_components)/,
            use: {
                loader: 'babel-loader',
                options: {
                presets: ['@babel/preset-env']
                }
            }
            }
        ]
    }

<!-- 第二种方式 -->
命令：npm install -D babel-loader@7 babel-core babel-preset-env webpack
版本：webpack 4.x | babel-loader 7.x | babel 6.x
配置：配置同上

##三、SASS

官网：
    https://github.com/webpack-contrib/sass-loader
命令：
    npm install sass-loader node-sass webpack --save-dev
    npm install style-loader css-loader --save-dev
版本：    
    "css-loader": "^1.0.1",
    "node-sass": "^4.10.0",
    "sass-loader": "^7.1.0",
    "style-loader": "^0.23.1",

配置：
    module: {
        rules: [
            {
                test: /\.css$/,
                use: [ "style-loader", "css-loader" ]
            },
            {
                test: /\.scss$/,
                use: [ "style-loader", "css-loader", "sass-loader" ]
            }
        ]
    }

##四、img

官网：
    https://github.com/webpack-contrib/file-loader
命令：
    npm install file-loader --save-dev
版本：
    "file-loader": "^2.0.0"
配置：
    rules: [
        {
            test: /\.(png|jpg|gif)$/,
            use: [
                {
                    loader: 'file-loader',
                    options: {
                        name: '[path][name].[ext]',
                        context: '',
                        publicPath: ''
                    }
                }
            ]
        }
    ]
使用：
    css中使用：
        background: url('./img/1.jpg');  //相对当前css文件
    js中使用：
        import imgName from './img/2.jpg';  //相对当前js文件
        img.src = imgName

##五：html热加载：src到dist编译过程的前半部分

思路：
    原来：src目录下只有一个index.js，编译后只有index.bundle.js
    现在：src目录下有index.js和another.js，编译后有app.bundle.js和another.bundle.js
配置：
    entry: {
        app: './src/index.js',
        another: './src/another.js'
    },
    output: {
        filename: '[name].bundle.js',
        path: path.resolve(__dirname, 'dist')
    },
使用：
    <!-- index.html中手动引入 -->
    <script src="./another.bundle.js"></script>
    <script src="./app.bundle.js"></script>

##六、html热加载：src到dist编译过程的后半部分

思路：
    原来：手动将诸如xx.bundle.js和yy.bundle.js添加进index.html
    现在：自动做上面的事情
    补充：
        自动覆盖掉原来的index.html中的内容，
        HtmlWebpackPlugin类中用template属性配置来解决此问题，
        把public目录下预先写好的index.html中的内容保存到dist目录下新生成的index.html中
官网：
    https://github.com/jantimon/html-webpack-plugin
命令：
    npm i --save-dev html-webpack-plugin@next
    npm i --save-dev html-webpack-plugin
配置：
    const HtmlWebpackPlugin = require('html-webpack-plugin');
    module.exports = {
        entry: {
            app: './src/index.js',
            print: './src/print.js'
        },
        plugins: [
          new HtmlWebpackPlugin({
              template: './public/index.html'
          })
        ],
        output: {
            filename: '[name].bundle.js',
            path: path.resolve(__dirname, 'dist')
        }
    };

##八、页面热更新：

1、使用观察者模式：

配置：
    "scripts": {
        ...
        "watch": "webpack --watch",
        ...
    },

2、使用webpack-dev-server服务器：

命令：
    npm install --save-dev webpack-dev-server
配置：
    module.exports = {
        ......
        devServer: {
            contentBase: path.join(__dirname, "dist"),
            compress: true,     // 开启gzip压缩
            port: 8888,         // 服务器端口号
            lazy: true,         // 惰性模式

        },
        ......
    };

    "scripts": {
        ...
         "start": "webpack-dev-server --open",
        ...
    },
使用：
    npm start
