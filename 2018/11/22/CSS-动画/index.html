CSS-动画

#参考资料：MDN文档

#一、好的习惯：

  1. 如果要灵活定制多个帧以及循环，用animation；
  2. 如果要简单的from to 效果，用 transition；
  3. 如果要使用js灵活设定动画属性，用transition.

#二、两个方案的不同点：

  • 触发：animation则和gif动态图差不多，立即播放，而transition需要一个事件去触发，如鼠标移 上去、焦点、点击等等，
  • 循环：animation可以设定循环次数；
  • 精确：animation可以设定每一帧的样式和时间，而tranistion 只能设定头尾；
  • 与javascript的交互：animation与js的交互不是很紧密，tranistion和js的结合更强大

#三、transform—-“改变”

属性值：

  • translate—-“平移”
    例：transform: translate( 100px, 200px ); //向右向下
  • scale—-“缩放”
    例：transform: scale( 0.5, 2 ); //变窄变高
  • skew—-“扭曲”
    例：transform: skew( 30deg, 60deg );
  • rotate—-“旋转”
    例：transform: rotate( 90deg ); //顺时针90度

补充：

###transform-origin—-“中心点”
例：transform-origin: right bottom;

属性值：left、right、center、length、%

#四、transition—-“过渡”
注意：除了两个时间外，顺序可以打乱

属性值：

  • transition-property —-“参与属性”
  • transition-duration —-“持续时间”
  • transition-delay —-“延迟时间”
  • transition-timing-function—-“动画节奏”
    • Linear—-“线性过渡”
    • ease —-“越来越慢”
    • ease-in —-“先慢后快”
    • ease-out —-“先快后慢”
    • ease-in-out —-“先慢后快再慢”

#五、animation—-“动画”
注意：除了两个时间外，顺序可以打乱

属性值：

  • animation-name —-“动画名称”
  • animation-duration —-“持续的时间”
    • s或ms为单位，没有单位值无效
  • animation-delay —-“延迟的时间”
  • animation-timing-function —-“动画节奏”
    • 和transition-timing-function一样的，此处省略
  • animation-direction —-“播放方向”
    • normal —-“次数为2，ease-in加快： 左到右–起点慢，左到右–起点慢”
    • alternate—-“次数为2，ease-in加快： 左到右–起点慢，右到左–起点慢”
    • reverse —-“次数为2，ease-in加快： 右到左–起点慢，右到左–起点慢”
    • alternate-reverse—“次数为2，ease-in加快： 右到左–起点慢，左到右–起点慢”
  • animation-iteration-count —-“播放次数”
    • number：默认是1；可以是小数，如0.5，表示播放动画到关键帧的一半（0 ~ 50%）
    • infinite—-“无限次播放”
  • animation-fill-mode —-“动画执行之前和之后如何给动画的目标应用样式”
    • none —- “动画前后样式不变”
    • forwards —-“停留在最后一帧”
    • backwards —-“停留在第一帧”
    • both —-“动画将会执行 forwards 和 backwards 执行的动作。”
  • animation-play-state —-“控制动画运行或暂停”
    • running—-“当前动画正在运行”
    • paused—-“当前动画已被停止”
    • 恢复一个已暂停的动画，将从它开始暂停的时候，而不是从动画序列的起点开始在动画。
