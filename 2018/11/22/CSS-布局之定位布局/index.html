CSS-布局之定位布局

#参考资料：《CSS权威指南》page：307页

#一、绝对定位：

  • 相对于包含块移动，偏移属性确定移动的距离；
  • 原位置删除，新位置不会嵌入其他元素的内容；（脱离文档流）
  • 块级元素生成块级框，行内元素也是生成块级框；（形成BFC）
  • 包含块：body元素或position不为static的祖先元素

属性：

  • top—-“下移”
  • bottom—-“上移”
  • left—-“右移”
  • right—-“左移”

#二、相对定位：

  • 相对于原位置移动，原位置保留，偏移属性确定移动的距离；
  • 块级元素生成块级框，行内元素生成行内框

属性：

  • top—-“下移”
  • bottom—-“上移”
  • left—-“右移”
  • right—-“左移”

#三、固定定位：

  • 相对于视窗移动，偏移属性确定移动的距离；（与absolute相似）
  • 原位置删除，新位置不会嵌入其他元素的内容；（脱离文档流）
  • 块级元素生成块级框，行内元素也是生成块级框（形成BFC）

属性：

  • top—-“下移”
  • bottom—-“上移”
  • left—-“右移”
  • right—-“左移”

#四、其他知识：

  • 根元素html的包含块是视窗大小的矩形
  • position样式值是relative或static时，包含块由最近的块级框、表单元格或行内块祖先框的内容边界构成
  • position样式值是absolute时，包含块由最近的position值不是static的祖先元素构成，
    当祖先元素是块级元素时，包含块是元素的内边距边界，
    当祖先元素是行内元素时，包含块是元素的内容边界。
  • 如果没有祖先元素，元素的包含块定义为初始包含块
  • 元素绝对定位后，会为其后代元素建立一个包含块
    （position不为static的都可做包含块）
  • 绝对定位元素相对于包含块移动，包含块是正常流的一部分，
    当文档流滚动时，包含块随之滚动，定位元素也随之滚动
  • 非替换元素的绝对定位时
    • left + margin-left + border-left + padding-left + width + padding-right + border-right + margin-right
      = 包含块的width
      • top + margin-top + border-top + padding-top +height + padding-bottom + border-bottom + margin-bottom
        = 包含块的height
